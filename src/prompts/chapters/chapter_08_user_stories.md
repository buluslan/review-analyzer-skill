## 八、典型用户深度解析（VOC）

### 核心要求

从高价值评论样本中提取最具代表性的用户，进行"原文 + 深度背景解析 + 核心需求探测"的连线分析。本章让冰冷的数据回归到有温度的真实用户故事。

### 选取标准

选取 4 个代表性用户，需满足以下分布要求：
- **情感分布均衡**：正面 2 个 + 负面 1 个 + 中立/混合 1 个
- **画像多样性**：尽量覆盖不同的人群画像
- **信息密度优先**：优先选取 info_score 高的评论
- **典型性**：选取能代表某一类用户群体的典型声音

### 解析框架

对每个用户，执行以下深度解析：

#### 1. 画像还原
根据评论内容和标签数据，还原该用户的基本画像。

#### 2. 决策链分析
还原该用户的购买决策过程：
- **触发点**：什么事件或需求触发了购买
- **评估过程**：用户关注了哪些维度、对比了什么
- **决策锚点**：最终促使其购买（或放弃复购）的关键因素
- **使用体验**：购买后的实际使用感受

#### 3. 隐性需求探测
从评论的字里行间挖掘用户没有直接表达但可以推断的需求。

### 头像类型

请根据评论风格从以下六类中选出一个最契合的头像类型：
- `male_young`（青年/中年男性）
- `female_young`（青年/中年女性）
- `male_elderly`（老年男性）
- `female_elderly`（老年女性）
- `child_boy`（少男/男童）
- `child_girl`（少女/女童）

### 输出格式

严格按以下格式输出每个用户的解析：

```
### 用户 [N]：[简短标签，如"追求性价比的通勤族"]

**典型画像**：[年龄段] [性别] [职业]
**头像类型**：[从六个枚举值中选择]
**核心需求**：[一句话概括，如：缓解长期久坐的腰部酸痛]
**情感倾向**：[正面/负面/中立]

**评价原文**：
> "[保留评论核心内容，控制在 100 字以内，过长可适当截断]"

**决策链分析**：
- 触发场景：[分析]
- 评估重点：[分析]
- 决策锚点：[分析]

**评论解析**：
[2-3 句话深度分析，指出该用户的爽点、痛点及隐性商业需求。不限于表面文字，需挖掘其背后的真实诉求]
```

### 写作规范

- 评论原文必须保留核心内容，不得仅显示 ID
- 每个用户的解析文字不少于 100 字
- 严禁编造评论内容，原文必须来自真实样本
- 解析需体现深度，不能是评论内容的简单复述
