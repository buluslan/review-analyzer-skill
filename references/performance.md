## 成本分析

### CLI 本地模式

**费用**: 免费（使用 Claude Code 配额）

| 项目 | 说明 |
|------|------|
| **显性费用** | $0 |
| **隐性费用** | 使用 Claude Code 配额 |
| **配额限制** | 受 Claude Code 使用额度限制 |

**适用场景**:
- ✅ 重要产品的深度分析
- ✅ 需要高质量洞察
- ✅ 单个产品或小批量分析
- ❌ 大批量分析（会耗尽配额）

### Gemini 增强模式

**费用**: 按使用量计费

#### 定价参考（以 Google 官方为准）

| 模型 | 输入 (每 1K tokens) | 输出 (每 1K tokens) |
|------|-------------------|-------------------|
| **Gemini 3.1 Flash** | $0.00015 | $0.00060 |
| **Gemini 3.1 Pro** | $0.00250 | $0.01000 |

#### 估算成本

假设分析 100 条评论，平均每条 50 词：

| 阶段 | 模型 | Tokens | 成本 |
|------|------|--------|------|
| **打标** | Gemini Flash | ~15,000 | ~$0.002 |
| **报告生成** | Gemini Flash | ~10,000 | ~$0.006 |
| **看板生成** | Gemini Pro | ~25,000 | ~$0.062 |
| **总计** | - | ~50,000 | **~$0.07** |

**成本范围**: $0.05 - $0.15 / 100 条评论

**影响因素**:
- 评论长度（越长越贵）
- 评论数量（批量有折扣）
- 模型选择（Pro 比 Flash 贵）

## 成本优化建议

### 1. 减少评论数量

使用 `--max-reviews` 参数控制分析的评论数：

```bash
# 只分析前 50 条
python3 main.py "reviews.csv" --max-reviews 50 --mode 1
```

**效果**:
- 成本线性降低
- 分析速度提升
- 适合初步筛选

### 2. 使用 Gemini Flash

Gemini Flash 比 Pro 便宜 10-20 倍，质量相近：

```bash
# 在代码中指定使用 Flash 模型
# （当前版本默认使用 Flash 生成报告）
```

**效果**:
- 成本降低 50-70%
- 速度提升 20-30%
- 质量轻微下降

### 3. 分批处理

将大文件分成多个小批次：

```bash
# 使用批量分析脚本
python3 run_batch.py --input-dir ./reviews --batch-size 50 --output-dir ./output
```

**效果**:
- 更好的成本控制
- 并行处理提升速度
- 便于错误恢复

### 4. 从已打标 CSV 生成报告

如果已有打标数据，直接生成报告：

```bash
python3 generate_from_tagged.py "tagged.csv" --mode 1 --creator "AI Assistant"
```

**效果**:
- 节省打标成本
- 只支付报告生成费用
- 适合二次分析

## 性能优化

### 1. 使用 Gemini Pro 生成看板

看板生成使用 Pro 模型，质量更高：

```python
# 在 config.py 中配置
HTML_GENERATION_MODEL = "gemini-3.1-pro-preview"
```

**效果**:
- 看板质量提升 20-30%
- 成本增加约 50%
- 适合重要产品

### 2. 并行处理

批量分析时使用多进程：

```bash
# 使用并行模式
python3 run_batch.py --input-dir ./reviews --parallel 4 --output-dir ./output
```

**效果**:
- 速度提升 3-4 倍
- 适合大批量分析
- 需要更多内存

### 3. 缓存优化

重复分析时使用缓存：

```bash
# 使用缓存模式
python3 main.py "reviews.csv" --cache-dir ./cache --mode 1
```

**效果**:
- 避免重复分析
- 节省成本和时间
- 适合迭代分析
